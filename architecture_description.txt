Для реализации веб-приложения была выбрана микросервисная архитектура с разделением на специализированные сервисы, каждый из которых отвечает за определенную область функциональности. В таком подходе клиентская часть запрашивает данные через API Gateway, который маршрутизирует запросы к соответствующим микросервисам. Компоненты общаются между собой, используя протокол HTTP и WebSocket для streaming-операций. На рис. 1 показано взаимодействие пользователя с приложением. На рис. 2 представлены основные компоненты приложения и их взаимодействие.

Приложение содержит следующие основные компоненты:

+ *Frontend (React + Three.js, порт 3000)* — клиентская часть с интерактивным 3D-редактором, чат-интерфейсом и панелью управления проектами. Взаимодействует с Backend Service через REST API и WebSocket;

+ *Backend Service (API Gateway, порт 8000)* — центральная точка входа, маршрутизирующая запросы к микросервисам. Обеспечивает валидацию JWT-токенов и проксирование запросов;

+ *Auth Service (порт 8001)* — сервис авторизации и аутентификации. Отвечает за регистрацию пользователей, подтверждение email, генерацию и валидацию JWT-токенов;

+ *Agents Service (порт 8006)* — сервис генерации 3D-контента на базе ML-модели ObLLoMov. Обрабатывает запросы на создание интерьеров, генерацию мебели и автоматическую расстановку объектов. Публикует события прогресса генерации в RabbitMQ для streaming обновлений;

+ *Chat Service (порт 8003)* — сервис AI-ассистента для диалогового взаимодействия. Обрабатывает текстовые запросы пользователя, парсит намерения и сохраняет историю диалогов;

+ *Project Service (порт 8004)* — сервис управления проектами пользователей. Реализует CRUD-операции для создания, редактирования, сохранения и удаления проектов интерьеров;

+ *Storage Service (порт 8005)* — сервис файлового хранилища. Обеспечивает загрузку и выгрузку 3D-моделей, текстур и других файлов в объектное хранилище MinIO/S3, генерирует presigned URLs для безопасного доступа;

+ *PostgreSQL (порт 5432)* — реляционная база данных для хранения пользователей, проектов, истории диалогов и метаданных;

+ *MinIO (порт 9000)* — S3-совместимое объектное хранилище для 3D-моделей, текстур и пользовательских файлов;

+ *RabbitMQ (порт 5672)* — очередь сообщений для асинхронной обработки задач генерации и передачи событий прогресса между сервисами;

+ *Redis (порт 6379)* — кеш для хранения сессий, временных данных и ускорения доступа к часто запрашиваемой информации.

Клиентская и серверная части разделены и взаимодействуют через REST API Backend Service, реализуя клиент-серверную архитектуру. Agents Service (генерация 3D) выделен в отдельный микросервис и общается с Backend Service через HTTP и RabbitMQ для streaming-генерации. Auth Service, Chat Service, Project Service и Storage Service также представляют собой независимые микросервисы, что обеспечивает масштабируемость и отказоустойчивость системы.
